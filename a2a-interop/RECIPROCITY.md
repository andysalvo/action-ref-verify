# Reciprocity Agreement

This fixture exchange operates on reciprocal verification:

1. **We emit, you validate.** Fixtures in `crest-produced/` are generated by action-ref-verify using the shared SHA-256(JCS(preimage)) derivation. We ask Agent OS (and any other implementation) to run them and report pass/fail.

2. **You emit, we validate.** Fixtures placed in `agent-os-pending/` will be consumed by our verifier. We will publish pass/fail results, including failures, in this repository.

3. **Failures are contributions.** A failing vector is more valuable than a passing one. It identifies a divergence that the spec needs to address. We will not suppress or hide failures.

4. **Attribution is preserved.** Every fixture carries its author and source. Cross-validation results credit both parties.

## How to submit fixtures

Place JSON files in `agent-os-pending/` following the format in `crest-produced/transactional-lifecycle-v0.json`:

```json
{
  "name": "your-vector-name",
  "spec": "action-ref-v1-jcs-sha256",
  "action_ref": "<expected SHA-256 hex>",
  "preimage": { ... },
  "notes": "What this tests"
}
```

Open a PR or drop the file and we will run it.

## Scope

This exchange covers the `transactional` claim_type for the v0.4 evidence-envelope format. Other claim_types can follow the same pattern in parallel directories.
